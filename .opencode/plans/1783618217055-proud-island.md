# Node.js Toolchain Refresh Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the entire build/release toolchain to use Node.js 26 runtime, latest Electron, and latest supporting tools.

**Architecture:** Monorepo with Bun workspaces (`packages/core`, `packages/cli`, `packages/desktop`). Build via `electron-vite`, package via `electron-builder`, CI via GitHub Actions.

**Tech Stack:** Node.js 26, Electron 44, TypeScript 6, Bun, electron-vite, electron-builder, Vite 8, React 19, Tailwind v4

## Global Constraints

- Node engine minimum: `>=26.0.0`
- Bun minimum: `>=1.2.0`
- TypeScript target: `ES2026`
- All desktop deps must be compatible with Electron 44 and Node 26

### Task 4: Update electron-vite Config for ESM and Chunking

**Files:**
- Modify: `packages/desktop/electron.vite.config.ts`

- [ ] **Step 1: Update the config**

Old content:
```ts
import path from 'node:path';
import { fileURLToPath } from 'node:url';

import { defineConfig } from 'electron-vite';
import react from '@vitejs/plugin-react';
import tailwindcss from '@tailwindcss/vite';

const currentDir = path.dirname(fileURLToPath(import.meta.url));
const coreEntry = path.resolve(currentDir, '../core/src/index.ts');

export default defineConfig({
  main: {
    resolve: {
      alias: {
        '@git-ingest/core': coreEntry,
      },
    },
    build: {
      externalizeDeps: {
        exclude: ['@git-ingest/core'],
      },
    },
  },
  preload: {},
  renderer: {
    plugins: [react(), tailwindcss()],
  },
});
```

New content:
```ts
import path from 'node:path';
import { fileURLToPath } from 'node:url';

import { defineConfig } from 'electron-vite';
import react from '@vitejs/plugin-react';
import tailwindcss from '@tailwindcss/vite';

const currentDir = path.dirname(fileURLToPath(import.meta.url));
const coreEntry = path.resolve(currentDir, '../core/src/index.ts');

export default defineConfig({
  main: {
    resolve: {
      alias: {
        '@git-ingest/core': coreEntry,
      },
    },
    build: {
      rollupOptions: {
        output: {
          format: 'es',
        },
      },
      externalizeDeps: {
        exclude: ['@git-ingest/core'],
      },
    },
  },
  preload: {
    build: {
      rollupOptions: {
        output: {
          format: 'es',
        },
      },
    },
  },
  renderer: {
    plugins: [react(), tailwindcss()],
    build: {
      rollupOptions: {
        output: {
          manualChunks: {
            react: ['react', 'react-dom'],
          },
        },
      },
    },
  },
});
```

- [ ] **Step 2: Verify the config is valid**

Run: `bun run --filter @git-ingest/desktop build`
Expected: Build succeeds with no errors

- [ ] **Step 3: Commit**

```bash
git add packages/desktop/electron.vite.config.ts
git commit -m "feat: enable ESM output and add chunking strategy in electron-vite config"
```

### Task 5: Update CI/CD Workflows

**Files:**
- Modify: `.github/workflows/ci.yml`
- Modify: `.github/workflows/release.yml`

- [ ] **Step 1: Update ci.yml**

Add Node.js setup step before the Bun setup step:

```yaml
- name: Setup Node.js
  uses: actions/setup-node@v4
  with:
    node-version: '26'
```

Insert this right after the checkout step and before the "Setup Bun" step.

- [ ] **Step 2: Update release.yml**

Add the same Node.js setup step in BOTH the `test` and `release` jobs, right after checkout and before setup-bun.

In the `release` job, also set `CSC_IDENTITY_AUTO_DISCOVERY: false` explicitly on the `env` for ALL matrix entries (currently it's only set on macOS via env on the Build package step). Move it to the job-level env:

```yaml
release:
  name: Build ${{ matrix.platform }}
  needs: test
  runs-on: ${{ matrix.os }}
  env:
    CSC_IDENTITY_AUTO_DISCOVERY: false
```

- [ ] **Step 3: Verify workflow syntax**

Run: `bunx action-validator .github/workflows/ci.yml && bunx action-validator .github/workflows/release.yml` (or just check YAML is valid: `bunx yaml-lint .github/workflows/ci.yml`)

- [ ] **Step 4: Commit**

```bash
git add .github/workflows/ci.yml .github/workflows/release.yml
git commit -m "ci: add Node 26 setup step to workflows"
```
