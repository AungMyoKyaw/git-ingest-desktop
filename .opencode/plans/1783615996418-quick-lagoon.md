# Release v0.2.0 Plan

## Overview

Release current `master` as v0.2.0. The existing `release.yml` workflow handles everything after a tag push.

## Steps

### 1. Bump version in all 4 package.json files

| File    | Path                            |
| ------- | ------------------------------- |
| Root    | `package.json`                  |
| Core    | `packages/core/package.json`    |
| CLI     | `packages/cli/package.json`     |
| Desktop | `packages/desktop/package.json` |

Change `"version": "0.1.0"` → `"version": "0.2.0"` in each.

### 2. Commit and tag

```bash
git add package.json packages/*/package.json
git commit -m "chore: bump version to 0.2.0"
git tag v0.2.0
git push origin master --tags
```

### 3. Monitor CI

Push triggers `release.yml` which:

1. **test** — runs all tests + smoke test + build
2. **release** — builds macOS (.dmg, .zip), Windows (.exe), Linux (.AppImage, .deb, .rpm)
3. **publish** — creates GitHub Release and uploads all artifacts

No manual intervention needed — artifacts are unsigned (ad-hoc signed on macOS, no notarization).
